# Anomaly Autoencoder (이상 탐지 오토인코더)

> 정상 데이터만으로 오토인코더를 학습한 뒤, 복원 오차가 임계값을 초과하는 샘플을 이상으로 판별하는 비지도 이상 탐지 모델

## 실행 방법

```bash
cd timeseries-ex/anomaly-autoencoder
python anomaly_autoencoder.py
```

의존성: `torch`, `numpy`

정상 500개(sin 파형 + 노이즈), 이상 50개(스파이크/드리프트/플랫) 합성 시계열을 사용한다. 이상 레이블은 학습에 사용되지 않는다.

## 핵심 개념

### 이상 탐지의 핵심 가정
오토인코더는 학습 데이터(정상)의 패턴만 복원할 수 있도록 수렴한다. 이상 데이터는 정상 분포를 벗어나므로 복원 오차가 크다.

```
정상 데이터:  복원 오차 작음 (오토인코더가 패턴을 익힘)
이상 데이터:  복원 오차 큼   (오토인코더가 패턴을 모름)
```

### 모델 구조
```
입력 (20) -> 인코더 (20->32->4) -> 잠재 표현 (4) -> 디코더 (4->32->20) -> 복원 출력 (20)
```

- **윈도우 크기**: 20 타임스텝
- **잠재 차원**: 4 (입력 대비 5배 압축)
- **손실**: MSE (Mean Squared Error)

### 임계값 설정
```python
threshold = normal_errors.mean() + 2 * normal_errors.std()
```

검증 셋(정상)의 복원 오차 분포에서 mean + 2*std를 임계값으로 사용한다. 정규분포 가정 시 약 95%의 정상 데이터를 포괄한다.

### 이상 유형
| 이상 유형 | 설명 | 복원 오차 |
|----------|------|---------|
| Spike | 특정 시점에 급격한 값 변화 | 매우 큼 |
| Drift | 선형적으로 증가하는 편향 | 큼 |
| Flat | 상수값 (패턴 소실) | 큼 |

### 평가 지표
- **FPR (False Positive Rate)**: 정상을 이상으로 잘못 탐지한 비율
- **Recall**: 실제 이상을 올바르게 탐지한 비율
- 임계값을 높이면 FPR 감소, Recall 감소 (트레이드오프)

## 관련 모델과의 비교

| 항목 | Autoencoder | Isolation Forest | One-Class SVM | LSTM-AE |
|------|-------------|-----------------|---------------|---------|
| 학습 방식 | 비지도 | 비지도 | 비지도 | 비지도 |
| 시계열 처리 | 윈도우 기반 | 독립 샘플 | 독립 샘플 | 시퀀스 직접 |
| 비선형 패턴 | 가능 | 제한적 | 커널 의존 | 가능 |
| 해석 가능성 | 복원 시각화 | 트리 경로 | 낮음 | 복원 시각화 |
| 계산 비용 | 중간 | 낮음 | 높음 | 높음 |

### Autoencoder vs VAE (Variational Autoencoder)
- **AE**: 결정론적 잠재 표현, 복원 품질 우선
- **VAE**: 확률적 잠재 표현, 생성 다양성 우선, 이상 탐지에 재구성 확률 사용 가능

### Autoencoder vs LSTM-AE
- **AE (MLP)**: 윈도우를 평탄화하여 처리, 시간 순서 무시
- **LSTM-AE**: 인코더/디코더에 LSTM 사용, 시간적 의존성 학습

### 임계값 전략 비교
| 전략 | 수식 | 특징 |
|------|------|------|
| 고정 백분위수 | 95th percentile | 데이터 분포에 강건 |
| Mean + k*Std | mean + 2*std (이 코드) | 정규 분포 가정 |
| ROC 최적화 | F1 최대화 임계값 | 레이블 데이터 필요 |
