# Reward Model (보상 모델)

> 인간 선호도 데이터로 Bradley-Terry 손실을 학습하여 RLHF의 기초가 되는 보상 신호를 생성하는 모델

## 실행 방법

```bash
cd safety-ex/reward-model
python reward_model.py
```

의존성: `torch`

합성 선호도 데이터 5쌍(도움됨·정확함·안전함·간결함·공손함 5개 특성)을 20배 증강하여 학습한다. 별도 설치 패키지 없이 PyTorch만으로 실행된다.

## 핵심 개념

### Bradley-Terry 선호도 모델
두 응답 중 어느 쪽이 더 나은지를 확률로 표현한다.

```
P(chosen > rejected) = sigmoid(r_chosen - r_rejected)
```

손실 함수는 이 확률의 음의 로그우도이다.

```
L = -log(sigmoid(r_chosen - r_rejected))
```

### 학습 구조
- **입력**: 응답의 특성 벡터 (5차원: 도움됨, 정확함, 안전함, 간결함, 공손함)
- **모델**: 3층 MLP (5 -> 32 -> 16 -> 1), 출력은 스칼라 보상 점수
- **학습 데이터**: (chosen, rejected) 쌍만 사용, 절대 레이블 불필요
- **데이터 증강**: 가우시안 노이즈(σ=0.1)로 5쌍을 100쌍으로 확장

### 보상 점수 해석
학습 후 동일 모델로 임의 응답을 채점할 수 있다.

| 응답 유형 | 예상 점수 |
|----------|---------|
| 도움 + 안전 | 높음 |
| 도움 + 불안전 | 낮음 |
| 도움 안 됨 + 안전 | 중간 |
| 도움 안 됨 + 불안전 | 매우 낮음 |

### RLHF에서의 역할
1. 인간 평가자가 응답 쌍에 선호도 레이블 부여
2. **보상 모델** 학습 (이 코드)
3. PPO 등 RL 알고리즘으로 LLM 정책 최적화
4. 보상 모델이 RL 환경의 보상 함수 역할 수행

## 관련 모델과의 비교

| 항목 | Reward Model (RLHF) | DPO |
|------|---------------------|-----|
| 학습 단계 | 2단계 (RM 학습 -> RL) | 1단계 (직접 최적화) |
| 별도 보상 모델 | 필요 | 불필요 |
| 안정성 | RL 불안정성 존재 | 안정적 (지도학습) |
| 유연성 | RM을 여러 정책에 재사용 가능 | 정책별 재학습 필요 |
| 대표 구현 | InstructGPT, ChatGPT | LLaMA-2-Chat |

### Reward Model vs Classifier
- **Classifier**: 절대 레이블(good/bad)로 학습, 클래스 불균형에 취약
- **Reward Model**: 상대적 선호도(A > B)로 학습, 인간 판단 자연스럽게 반영

### Reward Hacking
보상 모델의 한계: 정책이 보상 모델의 허점을 이용해 실제로는 나쁜 응답을 높은 점수로 유도할 수 있다. KL 페널티나 DPO로 완화한다.
